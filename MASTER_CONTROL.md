# MASTER CONTROL SYSTEM

## Purpose
This file acts as the single command interface for all operations.

All tasks start here.
All outputs pass through here.
All branches are controlled from here.

---

## SYSTEM FLOW

1. DEFINE TASK
2. CLASSIFY TASK
3. ASSIGN ENGINE
4. EXECUTE
5. AUDIT
6. RELEASE OR REVISE
7. LOG + STORE

## EXECUTION RULE

Before doing ANY task, answer:

1. What type is this?
2. Which branch handles it?
3. Which AI is best for it?
4. What is the final outcome?

---

## AUDIT GATE (MANDATORY)

Before release, confirm:

- does it solve the real problem?
- is it clean and structured?
- is it platform-correct?
- is it high-level (not generic)?
- would this pass recruiter or enterprise review?

If NO → STOP

---

## RELEASE CONDITIONS

Only release if:
- audited
- corrected
- aligned with goal
- formatted properly

---

## DAILY CLOSEOUT

At end of day:

- log work completed
- log errors
- log corrections
- define next action

---

## CORE RULE

No repetition without diagnosis.
No execution without classification.
No release without audit.
